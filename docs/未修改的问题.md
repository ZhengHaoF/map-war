## needsPlayerDecision 进事件层

**问题**：`needsPlayerDecision`（需玩家决策的停点）只在 `GameOrder` 指令层生效，未持久化到 `eventLog`。读档回放时不会在曾需要决策的位置暂停，玩家错过决策点。

**现状**：
- `GameOrder.needsPlayerDecision` 由 AI 返回（`aiOrderContract.ts` prompt 已描述该字段）
- 调度器 `advance()` 遇到后停止并吐 toast
- 但 `GameOrder` 不进 `eventLog`，读档时这些停点信息丢失

**修复方向**：
1. `game.ts`: 新增 `GameEvent` 类型 `{ type: 'pauseForPlayer' }`
2. `game.ts` `applyEvent`: 处理 `pauseForPlayer`，返回 `{ ok: true, paused: true }`
3. `game.ts` `load()`: 遇到 `paused` 事件时中断回放，暂存剩余事件，等玩家操作后继续
4. `useGameScheduler.ts`: 调度器遇到 `needsPlayerDecision` 时追加 `applyEvent({ type: 'pauseForPlayer' })` 写入日志
5. `game.ts`: 新增"继续回放"函数，玩家决策完成后回放剩余事件

**改动量**：约 21 行，不碰 UI / executeOrder / AI prompt

**状态**：待决定是否实施（2026-07-23）
