# 动画引擎设计文档

> **项目**：map-war（三国战略地图）  
> **技术栈**：Vue 3 + Vite + Pinia + PixiJS 8 + TypeScript  
> **文档版本**：v1.0  
> **最后更新**：2026-07-21

---

## 一、设计目标与约束

### 1.1 目标

- 为战略地图提供**电影级镜头语言**：聚焦、行军、侦察、宣战、战斗、占领均有独立视觉演出
- 保证 **AI 指令串行执行**：同一时刻只有一种动画在播放，避免并发冲突
- 读档后**战斗动画可恢复**：不丢失正在进行中的战斗视觉状态
- 遵循**无障碍标准**：支持 `prefers-reduced-motion` 降级

### 1.2 约束

- **不封装 PixiJS**：直接在 `troopAnimation.ts` 中调用 PixiJS v8 Graphics API
- **RAF 驱动，非 PixiJS Ticker**：项目使用原生 `requestAnimationFrame` / `cancelAnimationFrame` 驱动所有动画
- **动画与状态严格分离**：动画纯视觉，世界态变更统一通过 `useGameStore().applyEvent()` 落库
- **无统一 AnimationEngine 类**：采用函数式 + 轻量对象架构，每种动画为独立 async 函数

## 三、核心模块

### 3.1 动画 Primitives 层（`src/utils/`）

#### `troopAnimation.ts`（684 行）

**职责**： troops 动画 primitives，封装可复用的 async 动画函数

| 函数 | 类型 | 说明 |
|-----|------|------|
| `playArcAnimation` | async | 弧线飞行动画（dots/orb 模式），支持爆炸冲击波 + 弹字 |
| `playScoutAnimation` | async | 雷达扫描动画（旋转扇区 + 波环 + 接触点 + 信标脉冲） |
| `startBattleAnimation` | sync→返回 handle | 持续战斗光束（双向），需手动 `stop()` 清理 |
| `playCaptureAnimation` | async | 占领脉冲（城池点亮 + 双层冲击波 + 粒子爆发 + 弹字） |
| `showPopupText` | 内部 | 弹字飘字（requestAnimationFrame） |
| `drawFeatureOutline` | 内部 | 绘制 GeoJSON 轮廓高亮 |
| `drawArrow` | 内部 | 绘制三角形箭头 |

**关键模式**：
- 使用 `async/await + Promise` 包装 RAF 循环，使调用方能 `await` 动画完成
- 动画结束后自动 `removeChild` + `destroy` Graphics
- 所有坐标通过 `resolveLocationXY(id)` 解析

**关键接口**：

```typescript
export interface ArcAnimationOptions {
  fromId: string
  toId: string
  container: Container
  mode?: 'dots' | 'orb'
  explosion?: boolean
  shockwaves?: number
  text?: string
  textColor?: number
  color?: number
  dots?: number
  spacing?: number
  duration?: number
  explosionDuration?: number
}

export interface BattleHandle {
  stop: () => void
  graphics: Graphics | null
}
```

#### `cloudTransition.ts`（279 行）

**职责**：云雾蒙太奇（时间流逝转场）

| 函数 | 说明 |
|-----|------|
| `playCloudTransition` | 播放完整云雾动画（cover → hold → reveal） |
| `disposeCloudTransition` | 紧急清理（组件卸载时调用） |

**关键实现**：
- 使用 Canvas 2D 烘焙白云纹理（`getCloudTexture`），转为 PixiJS Texture
- 10+ 朵云从四边汇聚，使用 `lerp` + `easeInOut` 插值
- 中间点回调 `onMidpoint` 用于隐藏状态切换（如改日期）

**关键接口**：

```typescript
export interface CloudOptions {
  coverDuration?: number
  holdDuration?: number
  revealDuration?: number
  onMidpoint?: () => void | Promise<void>
}
```

### 3.2 编排层（`gameOrders.ts`，782 行）

**职责**：指令入口 + 动画编排 + 世界态唯一写者

**公开 API**：

| 导出符号 | 说明 |
|---------|------|
| `init(container, camera?, app?)` | 注入 PixiJS 容器/相机/app（setup 用） |
| `executeOrder(json)` | ★ 唯一指令入口，分发到各动画函数 |
| `playTimeJump(date)` | 时间跳跃演出（云雾蒙太奇 + 改日期） |
| `resetBattleRuntime()` | 清空战斗注册表（调试/重置用） |
| `restoreActiveAnimations()` | 读档后重建战斗动画（load 用） |

**指令类型（`OrderType`）**：

```typescript
export type OrderType =
  | 'arrowFly'      // 派兵/行军（黄点弧线）
  | 'radarPulse'    // 侦察（雷达扫描）
  | 'orbBurst'      // 宣战（红色光球+爆炸）
  | 'battle'        // 开战（双向交火光束）
  | 'stopBattle'    // 停战（单场）
  | 'stopBattles'   // 全线停战
  | 'listBattles'   // 查询战斗列表
  | 'fogCover'      // 云雾遮罩（转场）
  | 'capture'       // 占领城市
  | 'setFactionAlive' // 势力存亡
  | 'setCurrentDate'  // 推进日期
  | 'setCurrentFaction' // 切换视角势力
  | 'moveTroops'    // 调兵（复用 arrowFly 视觉）
```

**关键设计**：

1. **重入锁**：`locks.arrow/radar/orb/battle` 防止动画并发
2. **相机锁定**：`_camera.setLocked(true)` 防止用户抢相机
3. **镜头编排**：聚焦出发地 → 跟随到目的地 → 归位（三阶段）
4. **动画与状态分离**：动画播完后，由 `executeOrder` 统一 `applyEvent` 落库

**典型镜头编排**：

```typescript
// arrowFly 指令示例
const before = _camera.snapshot()
_camera.setLocked(true)
await _camera.focusOn(from, 600)
const follow = _camera.followTo(to, duration)
await playArcAnimation({ fromId: from, toId: to, container: _container, mode: 'dots', ... })
await follow
await _camera.reset(before)
_camera.setLocked(false)
```

### 3.3 PixiJS 舞台层（`LeafletMap.vue`）

**职责**：PixiJS 应用生命周期 + 容器分层 + 相机控制

**容器分层**：

| 容器 | 挂载点 | 用途 |
|-----|--------|------|

**相机系统**：

```typescript

  scale: number
  x: number
  y: number
}
```

**关键特性**：

## 四、动画类型详解

### 4.1 行军/派兵动画（`playArcAnimation`）

- **模式**：`dots`（点阵箭头）或 `orb`（光球）
- **路径**：二次贝塞尔曲线（`bezier(t, p0, p1, p2)`）
- **效果**：箭头沿弧线飞行 + 路径虚线 + 起点/终点轮廓高亮 + 到达后爆炸冲击波 + 弹字飘字
- **触发指令**：`arrowFly`、`moveTroops`

```typescript
// 贝塞尔曲线核心
function bezier(t: number, p0: Point, p1: Point, p2: Point): Point {
  const u = 1 - t
  return {
    x: u * u * p0.x + 2 * u * t * p1.x + t * t * p2.x,
    y: u * u * p0.y + 2 * u * t * p1.y + t * t * p2.y,
  }
}
```

### 4.2 侦察/雷达动画（`playScoutAnimation`）

- **效果**：同心波环扩散 + 旋转扫描扇区（拖尾余晖）+ 扫描前缘亮线 + 随机接触点光斑 + 中心信标脉冲
- **触发指令**：`radarPulse`

### 4.3 战斗动画（`startBattleAnimation`）

- **效果**：双向交火光束（A/B 两色箭头相向移动，循环播放）
- **生命周期**：常驻动画，需手动 `stop()` 清理
- **触发指令**：`battle`

### 4.4 占领动画（`playCaptureAnimation`）

- **效果**：城池轮廓脉冲点亮 + 双层冲击波扩散 + 8 枚菱形粒子爆发 + 弹字
- **缓动**：`easeOutCubic`（`1 - Math.pow(1 - t, 3)`）
- **触发指令**：`capture`

### 4.5 云雾蒙太奇（`playCloudTransition`）

- **效果**：多朵白云从四边汇聚盖满屏幕 → 停顿 → 退散揭开
- **用途**：时间流逝转场（`playTimeJump`）、状态切换遮罩
- **触发指令**：`fogCover`、`setCurrentDate`

### 4.6 镜头动画（`animateCameraTo`）

- **效果**：相机平滑补间（scale/x/y），`easeOutCubic` 缓动
- **场景**：聚焦城市、跟随行军、演出归位
- **触发点**：`gameOrders.ts` 中的各动画函数内部

### 4.7 缩放回弹（`scheduleZoomRest`）

- **效果**：用户缩放超出软边界（0.5x~8x）后，逐帧渐进回弹归位
- **特性**：可被下次滚轮/镜头补间打断

### 4.8 UI 过渡动画

- **Toast 入场/出场**：`ToastStack.vue` 使用 Vue `<TransitionGroup>`，下滑淡入/滑出淡出
- **Modal 展开**：`GameModal.vue` 使用 `<Transition>` 控制 opacity + scale
- **面板折叠**：`PlayerAiPanel.vue` / `AdvisorPanel.vue` 使用 `chat-collapse` 过渡

## 六、坐标空间设计

```
GeoJSON (经纬度) 
    ↓ geoToScreen()
世界坐标（未变换）
    ↓ worldContainer.scale/mapX/mapY
屏幕坐标
```

- **动画绘制在 `worldContainer` 内**，自动跟随相机平移/缩放
- **云雾蒙太奇挂在 `app.stage` 顶层**，不受相机影响，保证覆盖全屏
- **弹字反向缩放**：画在 `worldContainer` 内会随相机放大，通过 `textObj.scale.set(1 / container.scale.x)` 抵消，保持屏幕恒定大小

### 7.3 Cleanup 策略

| 场景 | 清理方式 |
|-----|---------|
| 单次动画（`playArcAnimation` / `playScoutAnimation` / `playCaptureAnimation`） | 动画结束后自动 `removeChild` + `destroy` Graphics |

| 组件卸载清理 | `disposeCloudTransition()` 清理云雾 |
| 动画重建 | `restoreActiveAnimations()` 从 store.battles 重建活跃战斗 |

### 7.4 读档恢复流程

```typescript
  restoreActiveAnimations()  // ← 从 store.battles 重建活跃战斗
```

## 八、与 PixiJS 的结合方式

### 8.1 封装层次：**半封装**

项目**没有完全封装 PixiJS**，而是**混合使用**：

| 层次 | 示例 | 说明 |
|-----|------|------|
| **原生 API 直接调用** | `new Graphics()`、`gfx.circle()`、`gfx.fill()`、`gfx.stroke()` |  troopAnimation.ts 中大量直接使用 PixiJS v8 API |
| **轻度封装** | `drawArrow()`、`drawFeatureOutline()` | 将 PixiJS 绘制命令封装为业务语义函数 |
| **容器层级封装** | `worldContainer`、`baseContainer`、`labelContainer` | LeafletMap 中分层管理，便于相机统一 transform |
| **应用生命周期管理** | `app.init()`、`app.stage.addChild()`、`app.destroy()` | LeafletMap 中管理 PixiJS Application 生命周期 |

## 九、缓动函数库

动画引擎使用以下缓动函数：

| 函数 | 公式 | 用途 |
|-----|------|------|
| `easeOutCubic` | `1 - Math.pow(1 - t, 3)` | 镜头归位、占领脉冲 |
| `easeInOutCubic` | `t < 0.5 ? 4t³ : 1 - (-2t+2)³/2` | 云雾蒙太奇 |
| `rubberbandClamp` | 越界后增量衰减 | 缩放软边界 |

**无第三方动画库**：项目不引入 GSAP、anime.js 等库，所有缓动和插值手写实现。

## 十、设计亮点

1. **动画与状态严格分离**：动画纯视觉，世界态变更统一通过 `applyEvent` 落库
2. **重入锁防并发**：每种动画类型独立锁，避免并发播放冲突
3. **读档可恢复**：`restoreActiveAnimations()` 从 store 重建活跃战斗动画
4. **无障碍降级**：`prefersReducedMotion()` 检测，关闭镜头补间
5. **ESC 中断**：镜头演出和缩放回弹均可被 ESC 取消
6. **模块级单例**：`useGameScheduler` / `useAgentKernel` / `useToast` 均为模块级单例，跨组件共享状态
7. **镜头三阶段编排**：聚焦出发地 → 跟随行军 → 归位，形成完整的视觉叙事
8. **云雾蒙太奇藏状态切换**：`onMidpoint` 回调在云雾盖满屏幕时执行状态变更，玩家感知不到中间过程

## 十一、已知限制与 TODO

- **无统一的 AnimationEngine 类**：当前动画能力分散在多个函数中，缺乏统一的调度和管理接口
- **无时间轴/关键帧系统**：所有动画为单次 RAF 循环，不支持复杂的时间轴编排
- **无动画优先级队列**：重入锁机制简单，不支持动画排队或优先级抢占
- **PixiJS 句柄与 Store 未完全解耦**：`battleRegistry` / `activeBattles` 为模块级变量，存在潜在内存泄漏风险
- **无性能监控**：未实现 FPS 监控、动画帧时间统计等性能指标
