## 二、阻断性问题（必须修，否则 `build`/`type-check` 全挂）

### 1. vue-tsc 与 typescript 版本不兼容（工具链崩）

- **现象**：`npm run type-check` 和 `npm run build` 直接抛
  `ERR_PACKAGE_PATH_NOT_EXPORTED: Package subpath './lib/tsc' is not defined ... typescript/package.json`
- **原因**：`package.json` 中 `typescript@^7.0.2` 与 `vue-tsc@3.3.7` 不兼容。vue-tsc 3.x 仍 `require('typescript/lib/tsc')`，而 TS7 已移除该子路径导出。
- **影响**：类型检查与构建脚本**完全无法运行**，迁移验收标准里「type-check 零错误」目前无法满足。
- **建议（沙箱无网络，我未能替你安装）**：把 `typescript` 降到 `~5.6`、`vue-tsc` 降到 `~2.x`（已知兼容组合）后重新安装；或升级 vue-tsc 到支持 TS7 的版本。装好后请复跑 `npm run type-check`。

### 2. `tsconfig.json` 的 `include` 漏了 `.vue` —— 已修复

- **现象**：`include` 原先只有 `["src/**/*.ts","src/**/*.d.ts"]`，没有 `.vue`。
- **影响**：即便 vue-tsc 修好，Vue 组件也**完全不参与类型检查**（等于白迁移）。
- **已处理**：已加入 `"src/**/*.vue"`。

## 三、运行时崩溃 bug —— 已修复

### `GameModal.vue`：`props` 未赋值

- **位置**：`src/components/ui/GameModal.vue`
- **问题**：`withDefaults(defineProps<...>(), {...})` **裸调用**，返回值没赋给 `props`，但脚本里却用了 `props.initX / props.minWidth / props.draggable`。
  在 `<script setup>` 中，`defineProps` 的返回值**必须赋值**才能在脚本里用 `props.xxx`（模板里能直接用名字，脚本里不行）。不赋值则 setup 时 `props` 为 `undefined` → 组件挂载即抛 `Cannot read properties of undefined`。
- **影响**：所有 `GameModal` 使用处（信息弹窗、调试面板、战斗管理、免责声明）都会崩。
- **已修复**：改为 `const props = withDefaults(defineProps<...>(), {...})`。

## 四、会让 vue-tsc 报错的真实类型问题 —— 已修复（待 vue-tsc 跑通后复验）

以下都在 `src/components/LeafletMap.vue`，运行时目前不崩（逻辑本身正确），但类型不匹配会让 vue-tsc 报错、从而卡住 `build`：

1. **`drawFeature` / `highlightOn` / `highlightBaseFeature` 的 `polygons` 层级写浅了**
   `const polygons: number[][][]` 但实际 Polygon 分支 `[geometry.coordinates]` 是 4 层 `number[][][][]`（MultiPolygon 分支也是 4 层）→ vue-tsc 报 `Type 'number[][][][]' is not assignable to 'number[][][]'`。
   → 已改为 `number[][][][]`（运行时不变）。

2. **`loadLayer` 给 `drawFeature` 多传了第 6 个参数 `mapScale`**
   调用：`drawFeature(graphics, feature, width, height, {...}, mapScale)`，函数只有 5 个形参 → vue-tsc 报 `Expected 5 arguments, but got 6`。
   → 已删除多余 `mapScale`（坐标缩放本由 `worldContainer` 的 transform 负责，drawFeature 不需要它）。

3. **`infoRows` 索引键可能为 `undefined`**
   `OWNER_NAMES[d.owner]` / `TERRAIN_NAMES[d.terrain]` / `LEVEL_NAMES[d.cityLevel]` 的键在 strict 下可能是 `undefined` → 报索引类型错误。
   → 已加兜底：`d.owner ?? ''`、`d.terrain ?? ''`、`d.cityLevel ?? 0`。

4. **`countryInfoRows` 的 `value` 推导为 `unknown`**
   `infoCountryData` 类型是 `Record<string, unknown>`，其字段在 `value:` 中推导为 `unknown`，而 `InfoTable` 的 `Row.value` 是 `string | number` → 不兼容（特别是 `全称` 那行 `d.full_name || '—'`）。
   → 已改为 `(d.full_name as string) || '—'`（其余字段已用模板字符串包成 string）。
