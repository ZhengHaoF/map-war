# AI 游戏指令参考

> 本文档供 AI 决策时参考，描述了 AI 可以执行的指令、可用数据、约束规则和返回格式。

## 一、游戏概述

这是一款以 **1931 年九一八事变前后**为背景的中国战争 AI 游戏。地图使用 PixiJS 渲染，AI 通过返回 JSON 指令来控制游戏中的各种行动。所有行动都会在地图上播放相应的动画效果。

## 二、地点 ID 系统

AI 在执行任何指令时，都需要使用地点 ID。地点分为两类：

### 2.1 中国城市（使用行政区编码 gb）

- **格式**：9 位数字字符串，例如 `"156500000"`（上海市）、`"156450200"`（南京市）
- **唯一标识**：`gb` 字段，与 GeoJSON 数据的 `properties.gb` 对应
- **范围**：覆盖中国约 370 个城市（省级、市级、县级）
- **说明**：gb 编码前 2~6 位表示省/直辖市，后续表示市/县

### 2.2 世界国家（使用 ISO 3166-1 alpha-3 代码）

- **格式**：3 位大写字母字符串，例如 `"JPN"`（日本）、`"SUN"`（苏联）、`"USA"`（美国）
- **唯一标识**：`iso_a3` 字段，与 GeoJSON 数据的 `properties.iso_a3` 对应
- **特殊映射**：现代代码 `RUS` 会自动映射为游戏代码 `SUN`（苏联）
- **范围**：覆盖全球约 15 个主要国家

### 2.3 如何获取地点 ID

AI 可以通过以下方式获取可用地点列表：

| 数据类型 | 获取方式 | 说明 |
|----------|----------|------|
| 中国城市 | 从 `chinaCities` 数据中读取 | 每个城市有 `gb`、`name`、`owner` 等字段 |
| 世界国家 | 从 `worldCountries` 数据中读取 | 每个国家有 `iso_a3`、`name`、`diplomacy` 等字段 |

## 三、AI 指令协议

AI 只需要返回一个 JSON 对象，系统会自动解析并执行对应的游戏指令。

### 3.1 指令总览

| 指令 | 用途 | 动画效果 | 默认文字 |
|------|------|----------|----------|
| `attack` | 派兵/出兵 | 多箭头沿弧线飞行 | "出兵！" |
| `scout` | 侦察/探察 | 绿色扩散圆环 | "侦察！" |
| `declareWar` | 宣战 | 光球+爆炸+冲击波 | "宣战！" |
| `battle` | 持续战斗 | 双向箭头循环交火 | 无 |
| `stopBattle` | 停止指定战斗 | 清除动画 | - |
| `stopBattles` | 停止所有战斗 | 清除所有动画 | - |
| `listBattles` | 查询当前战斗列表 | 无 | - |

### 3.2 attack — 派兵

派兵从起点城市出发，飞往目标城市/国家。

```json
{
  "order": "attack",
  "from": "156500000",
  "to": "156450200",
  "text": "猛攻！"
}
```

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `order` | 是 | string | 固定值 `"attack"` |
| `from` | 是 | string | 起点地点 ID（城市 gb 或国家 iso_a3） |
| `to` | 是 | string | 终点地点 ID |
| `text` | 否 | string | 到达后弹出的文字，默认 "出兵！" |

**动画参数**：
- 模式：多箭头（5 个）
- 颜色：金色 `0xffcc00`
- 时长：2000ms
- 路径：二阶贝塞尔弧线

**约束**：
- 同一时间只能执行一次派兵动画（互斥锁）
- 如果派兵动画正在进行中，返回 `{ ok: false, reason: "派兵动画进行中" }`

**返回值**：
```json
{ "ok": true }
```
或
```json
{ "ok": false, "reason": "派兵动画进行中" }
```

### 3.5 battle — 持续战斗

在两个城市/国家之间发起持续交火动画。

```json
{
  "order": "battle",
  "from": "156500000",
  "to": "156450200"
}
```

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `order` | 是 | string | 固定值 `"battle"` |
| `from` | 是 | string | 己方地点 ID |
| `to` | 是 | string | 敌方地点 ID |

**动画参数**：
- 模式：双向箭头持续循环
- 己方颜色：蓝色 `0x3b82f6`（3 个箭头）
- 敌方颜色：红色 `0xef4444`（3 个箭头）
- 路径：二阶贝塞尔弧线 + 白色虚线路径
- 循环：到达终点后自动重置，持续播放直到手动停止

**约束**：
- **同一对 from → to 不能重复发起**。如果已存在该对战斗，返回 `{ ok: false, reason: "已存在 xxx → yyy 的战斗" }`
- 每个战斗分配唯一 ID（格式：`battle_1`、`battle_2`...），用于后续管理

**返回值**：
```json
{ "ok": true, "id": "battle_1" }
```
或
```json
{ "ok": false, "reason": "已存在 156500000 → 156450200 的战斗" }
```

### 4.3 AI 战略价值公式

AI 可以参考以下公式计算城市战略价值：

```
attackScore = cityLevel * 3 + industry * 2 + food - fort
```

数值越高，越值得进攻。地形会直接影响攻防修正。

## 五、约束与规则

### 5.1 动画互斥锁

| 指令类型 | 互斥锁 | 说明 |
|----------|--------|------|
| `attack` | `attack` | 同一时间只能执行一次派兵 |
| `scout` | `scout` | 同一时间只能执行一次侦察 |
| `declareWar` | `war` | 同一时间只能执行一次宣战 |
| `battle` | 无 | 可以同时发起多场战斗（不同对） |

不同类型指令之间**互不干扰**，可以同时执行（如派兵 + 侦察 + 宣战并行）。

### 5.2 战斗去重

- 同一对 `from → to` 的城市/国家只能存在一场战斗
- 如果尝试重复发起，会返回错误
- 必须先 `stopBattle` 停止旧战斗，才能重新发起

### 5.3 地点有效性

- 所有 `from`、`to` 字段必须是已知的地点 ID
- 如果 ID 无效，动画无法解析坐标，会静默返回（控制台有警告）
- AI 应优先使用从 `chinaCities` 和 `worldCountries` 中读取的有效 ID

### 5.4 异步执行

- `attack`、`scout`、`declareWar` 是异步指令，返回 Promise
- `battle`、`stopBattle`、`stopBattles`、`listBattles` 是同步指令
- `executeOrder` 统一封装，自动处理同步/异步

## 七、错误处理

| 错误原因 | 说明 |

| `派兵动画进行中` | 上一次派兵动画尚未完成 |
| `探察动画进行中` | 上一次侦察动画尚未完成 |
| `宣战动画进行中` | 上一次宣战动画尚未完成 |
| `gameOrders 未初始化` | 系统尚未完成初始化 |

## 九、AI 决策建议

### 9.1 选择进攻目标

AI 应综合考虑以下因素：

1. **城市价值**：`cityLevel * 3 + industry * 2 + food - fort`
2. **地形优势**：山地/森林防御加成高，平原易攻难守
3. **当前控制者**：弱小政权的城市更容易攻占
4. **距离**：相邻城市更容易补给和支援

### 9.2 侦察优先级

- 对未知区域或敌方控制区域优先侦察
- 侦察可以获取目标城市信息（虽然当前版本主要是视觉效果）

### 9.3 战斗管理

- 战斗是持续动画，不需要重复发起
- 重要战斗可以持续展示，次要战斗应及时停止
- 可以使用 `listBattles` 定期检查战斗状态

### 9.4 宣战时机

- 宣战是强烈的信号，建议在发起进攻前宣战
- 宣战后再 `attack` 或 `battle` 形成完整的战争流程

## 十、当前版本限制

- **无持久化**：所有战斗状态在页面刷新后丢失
- **无胜负判定**：战斗动画仅为视觉效果，不改变城市控制权
- **无部队消耗**：派兵/战斗不消耗兵力或资源
- **无补给系统**：暂未实现部队补给和士气系统
- **单玩家**：当前为单人演示，AI 控制所有指令
