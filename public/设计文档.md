# 中国战争 AI 游戏 — 城市数据规范

> Version: 1.0

---

## 一、设计理念

城市是整个游戏的**战略节点（Strategic Node）**。

城市不负责保存所有数据，而只保存**长期稳定且具有战略价值的数据**。

例如：

- 城市规模
- 工业能力
- 粮食生产能力
- 地形
- 防御能力

而下面这些属于动态数据，不保存在城市：

- 部队
- 补给
- 士气
- 天气
- 占领进度
- 建筑施工
- 战斗状态

这些数据由其它系统维护。

这样做可以让：

- 数据更加清晰
- AI 更容易计算
- 剧本更容易制作
- 后期更容易扩展

---

## 二、城市数据结构

```ts
interface City {
  /** 行政区编码（唯一ID） */
  gb: string
  /** 城市名称 */
  name: string
  /** 当前实际控制政权 */
  owner: Owner
  /** 地形 */
  terrain: Terrain
  /** 城市规模 */
  cityLevel: number
  /** 工业能力 */
  industry: number
  /** 粮食生产能力 */
  food: number
  /** 工事等级 */
  fort: number
}
```

---

## 三、字段说明

| 字段        | 类型     | 范围     | 说明                 |
| ----------- | -------- | -------- | -------------------- |
| gb          | String   | 唯一     | 行政区代码，唯一ID   |
| name        | String   | -        | 城市名称             |
| owner       | Enum     | -        | 当前实际控制政权     |
| terrain     | Enum     | -        | 城市地形             |
| cityLevel   | Number   | 1~5      | 城市综合规模         |
| industry    | Number   | 0~10     | 工业能力             |
| food        | Number   | 0~10     | 粮食生产能力         |
| fort        | Number   | 0~5      | 工事等级             |

---

## 四、Owner（政权）

```ts
enum Owner {
  KMT,  // 国民政府
  CCP,  // 中共苏区
  JPN,  // 日本关东军
  NEA,  // 东北军
  SHX,  // 晋系
  GXC,  // 桂系
  SCC,  // 川军
  MA,   // 马家军
  XJ,   // 新疆
  TIB   // 西藏
}
```

**说明：** `owner` 表示当前城市的**实际控制者**。

例如，1931年：

| 城市 | owner |
| ---- | ----- |
| 沈阳 | JPN   |
| 太原 | SHX   |
| 长沙 | KMT   |

---

## 五、Terrain（地形）

```ts
enum Terrain {
  PLAIN,
  HILL,
  MOUNTAIN,
  FOREST,
  CITY
}
```

**说明：** 地形影响行军速度、战斗攻防、AI寻路、AI战略价值。

**推荐战斗修正：**

| 地形       | 攻击  | 防御  | 行军  |
| ---------- | ----- | ----- | ----- |
| PLAIN      | 100%  | 100%  | 100%  |
| HILL       | 90%   | 120%  | 90%   |
| MOUNTAIN   | 70%   | 150%  | 60%   |
| FOREST     | 80%   | 130%  | 70%   |
| CITY       | 90%   | 140%  | 80%   |

## 六、城市规模（cityLevel）

城市规模表示：人口、经济、政治影响力、交通地位的**综合评价**（不是人口）。

**范围：** 1~5

**评分标准：**

| 等级 | 类型       |
| ---- | ---------- |
| 1    | 县城、小城 |
| 2    | 普通城市   |
| 3    | 区域中心   |
| 4    | 全国重要城市 |
| 5    | 超级城市   |

**参考：**

| ---- | --------- |
| 上海 | 5         |
| 北京 | 5         |
| 南京 | 5         |
| 武汉 | 4         |
| 广州 | 4         |
| 重庆 | 4         |
| 天津 | 4         |
| 长沙 | 3         |
| 南昌 | 3         |
| 济南 | 3         |
| 桂林 | 2         |
| 阿坝 | 1         |

## 十、AI 推荐使用方式

AI 可以直接根据城市数据计算战略价值。

```ts
attackScore = cityLevel * 3 + industry * 2 + food - fort
```

数值越高，越值得进攻。

**示例：**

| 城市 | 计算                  | 得分 |
| ---- | --------------------- | ---- |
| 上海 | 5×3 + 10×2 + 2 - 1   | 36   |
| 长沙 | 3×3 + 4×2 + 7 - 2    | 22   |

AI 将优先进攻上海。

## 十一、设计原则

### 1. 统一量纲

整个游戏尽量采用统一评分体系，方便 AI 权重计算。

| 属性      | 范围  |
| --------- | ----- |
| cityLevel | 1~5   |
| industry  | 0~10  |
| food      | 0~10  |
| fort      | 0~5   |

### 2. 城市保存静态数据

| 保存     | 不保存   |
| -------- | -------- |
| 地形     | 部队     |
| 工业     | 补给     |
| 粮食     | 士气     |
| 城市规模 | 天气     |
| 工事     | 战斗状态 |

### 3. 所有字段必须参与 AI 决策

城市中的每一个字段，都必须直接影响 AI：

- 工业高 → AI 优先占领
- 粮食高 → AI 优先建立补给中心
- 工事高 → AI 优先绕路或集中兵力
- 城市规模高 → AI 优先死守
