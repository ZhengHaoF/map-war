# 开发规范

## 项目概述

本项目是一个中国战争 AI 游戏，使用 Vue 3 + Pixi.js 实现地图交互。

---

### GameModal

弹窗/浮动面板组件，自动 Teleport 到 body。支持普通居中弹窗和可拖动的浮动面板两种模式。

```vue
<!-- 普通弹窗（居中、带遮罩） -->
<GameModal :visible="showModal" title="标题" @close="showModal = false">
  <p>内容</p>
</GameModal>

<!-- 可拖动的浮动面板（无遮罩，自由定位） -->
<GameModal
  :visible="showPanel"
  title="面板"
  :draggable="true"
  :overlay="false"
  :init-x="160"
  :init-y="160"
  @close="showPanel = false"
>
  <p>可拖动的面板内容</p>
</GameModal>
```

| Prop | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| visible | Boolean | false | 是否显示 |
| title | String | `''` | 标题文本 |
| minWidth | String | `'280px'` | 最小宽度 |
| draggable | Boolean | false | 是否可拖动 |
| overlay | Boolean | true | 是否显示遮罩层 |
| initX | Number | 160 | 初始 X 位置（`draggable` 时生效） |
| initY | Number | 160 | 初始 Y 位置 |

| Event | 说明 |
|-------|------|
| close | 关闭弹窗 |

**拖动说明：** 按住标题栏拖拽，标题栏在 `draggable` 时显示 `grab/grabbing` 光标。`overlay=false` 时遮罩透明且不阻塞地图交互。

## 图标库（Tabler Icons）

本项目使用 **unplugin-icons + @iconify-json/tabler** 作为图标方案，编译期按需导入、零运行时体积。

### 安装

已在 devDependencies 中安装，无需额外操作：

```json
"devDependencies": {
  "unplugin-icons": "^23.0.1",
  "@iconify-json/tabler": "^1.2.35"
}
```

### 静态用法（推荐：图标名固定）

用 `~icons/tabler/{图标名}` 显式导入组件，然后通过 `<component :is>` 渲染：

```vue
<script setup lang="ts">
import IconSword from '~icons/tabler/sword'

// 也可通过查表动态使用
import IconFlag from '~icons/tabler/flag'
import IconX from '~icons/tabler/x'
import type { Component } from 'vue'

const ICONS: Record<string, Component> = {
  flag: IconFlag,
  x: IconX,
}
</script>

<template>
  <!-- 直接引用 -->
  <component :is="IconSword" :size="16" />

  <!-- 通过查表动态切换 -->
  <component :is="ICONS['flag']" :size="16" />
</template>
```

> ⚠️ **不要使用 `<i-tabler-xxx>` 标签写法。** `unplugin-icons` 的 `compiler: 'vue3'` 选项**不会在运行时自动注册全局组件**，Vue 会将其当作未知自定义元素处理，导致图标不渲染。必须用 `~icons/tabler/xxx` 显式 import 后通过 `<component :is>` 渲染。

### 动态用法（图标名来自数据/配置）

右键菜单等需要根据数据字段动态渲染图标时，先静态导入所有用到的图标，再通过查表映射：

```vue
<script setup lang="ts">
import type { Component } from 'vue'
import IconInfoCircle from '~icons/tabler/info-circle'
import IconSearch from '~icons/tabler/search'

const ICONS: Record<string, Component> = {
  'info-circle': IconInfoCircle,
  search: IconSearch,
}
</script>

<template>
  <component :is="ICONS[item.icon]" :size="16" />
</template>
```

### 已用图标一览

Tabler Icons 全部图标可在 https://tabler.io/icons 搜索。组件导入路径为 `~icons/tabler/{图标名}`（kebab-case）：

| 语义 | import 路径 | 用途 |

### 配置说明

`vite.config.ts` 中的插件配置：

```ts
import Icons from 'unplugin-icons/vite'

export default defineConfig({
  plugins: [
    vue(),
    vueDevTools(),
    Icons({ compiler: 'vue3' }),
  ],
})
```

```ts
/// <reference types="unplugin-icons/types/vue3" />  // 全局组件类型
declare module '~icons/*' {                          // 显式 import 类型
  import type { DefineComponent } from 'vue'

  export default component
}
```

### 世界国家数据

定义在 `src/data/worldCountries.ts`：

```ts
interface CountryData {
  /** 唯一ID，与 iso_a3 相同 */
  id: string
  /** ISO 国家代码（与世界.geojson 对应） */
  iso_a3: string
  /** 中文国名 */
  name: string
  /** 中文全称 */
  full_name: string
  /** 国家类型 */
  countryType: 'EMPIRE' | 'REPUBLIC' | 'UNION' | 'COLONY' | 'KINGDOM' | 'SPLIT'
  /** 军事实力 (0-10) */
  military: number
  /** 工业能力 (0-10) */
  industry: number
  /** 人口/资源 (0-10) */
  population: number
  /** 对华威胁等级 (0-10) */
  threat: number
  /** 外交关系 */
  diplomacy: 'ALLIED' | 'FRIENDLY' | 'NEUTRAL' | 'HOSTILE' | 'WAR'
}
```

**国家类型说明：**

| 值 | 含义 |
|----|------|
| EMPIRE | 帝国（日本、英国） |
| REPUBLIC | 共和国（美国、法国） |
| UNION | 联盟/联邦（苏联） |
| COLONY | 殖民地（印度、越南） |
| KINGDOM | 王国（泰国、阿富汗） |
| SPLIT | 分裂/内战 |

**外交关系说明：**

| 值 | 含义 | 地图着色 |
|----|------|----------|
| ALLIED | 同盟 | 深蓝 `0x1a3a7e` |
| FRIENDLY | 友好 | 暗蓝灰 `0x2a3a5e` |
| NEUTRAL | 中立 | 暗橄榄 `0x2d3a2e` |
| HOSTILE | 敌对 | 暗红 `0x6b2020` |
| WAR | 交战中 | 深红 `0x4b0000` |

**AI 威胁评估公式：**

```
countryThreat = military * 3 + industry * 2 + population + diplomacyOffset
```

| 外交关系 | diplomacyOffset |
|----------|-----------------|
| ALLIED | -10 |
| FRIENDLY | -5 |
| NEUTRAL | 0 |
| HOSTILE | +5 |
| WAR | +10 |

## 状态管理（useGameStore）

全局游戏状态统一由 Pinia store `src/stores/game.ts` 的 `useGameStore()` 管理，作为唯一真相源。
store 只持有「全局要用的变量」，不含推演逻辑。

### 读：getSnapshot（放 store 内）

AI 需要读取当前局势时，调用 `useGameStore().getSnapshot()`，返回一份**脱离响应式**的纯对象快照，
可直接 `JSON.stringify` 序列化后传给 LLM，且不会在调用方建立响应式依赖。

```ts
const state = useGameStore().getSnapshot()
// {
//   currentDate: string,
//   currentFaction: Owner | null,
//   activeFactions: Owner[],
//   ownership: Record<string, Owner>,   // gb -> 控制政权
//   battles: BattleInfo[]
// }
```

> 注意：快照是拷贝，调用方对其修改不影响 store。

### 写：最小参数 setter（放 gameOrders.ts）

AI 指令「落地」回写 store 时，走 `src/utils/gameOrders.ts` 提供的 setter，只接收最少必要参数，
不直接碰 PixiJS / 动画。

```ts
import {
  setCityOwner,
  setFactionAlive,
  setCurrentDate,
  setCurrentFaction,
} from '@/utils/gameOrders'

setCityOwner('156450200', Owner.JPN)   // 占领：城市 gb -> 新政权
setFactionAlive(Owner.TIB, false)      // 势力灭亡：从存活列表移除
setCurrentDate('1931-10-01')           // 推进日期时钟
setCurrentFaction(Owner.KMT)           // 选玩家势力（委托 store.selectFaction）
```

| 方法 | 参数 | 说明 |
|------|------|------|
| setCityOwner | (gb, owner) | 设置/变更某城市控制政权 |
| setFactionAlive | (faction, alive) | 设置势力存活：alive=false 即灭亡 |
| setCurrentDate | (date) | 推进全局日期（ISO 字符串） |
| setCurrentFaction | (faction) | 设置玩家势力 |

### 设计约定

- **读放 store、写放 gameOrders**：读接口作为 store action（`getSnapshot`），写接口作为指令层 setter，职责清晰。
- **store 是纯变量容器**：不含事件推演引擎；推演逻辑后续独立模块实现，再回写这些 setter。
- **battles 仅存元数据**：`BattleInfo`（id/from/to/names/active），PixiJS 动画句柄留在 gameOrders 本地，不进 store。

---

## 开发约定

1. **组件命名**：通用组件以 `Game` 前缀命名
2. **样式隔离**：使用 `scoped` 避免样式污染
3. **颜色格式**：PixiJS 使用 `0xRRGGBB`，CSS 使用 `#RRGGBB`
4. **事件命名**：使用 kebab-case（如 `@select`、`@close`）
5. **Prop 类型**：使用对象形式定义，包含 type、default、required

## 动画函数库

文件位置：`src/utils/troopAnimation.js`

### playArcAnimation - 弧线动画（通用）

从起点到终点的弧线动画，支持两种模式和文字弹出。

**模式：**
- `mode: 'dots'` - 多箭头沿弧线移动（适合派兵、行军）
- `mode: 'orb'` - 光球沿弧线移动（适合宣战、施法）

**效果组合：**
- `explosion: true` - 到达后爆炸+冲击波
- `text: '文字'` - 到达后在目的地上方弹出飘字

```js
import { playArcAnimation } from '@/utils/troopAnimation'

// 派兵模式（箭头 + 飘字）
await playArcAnimation({
  from: { x: 500, y: 300 },
  to: { x: 600, y: 400 },
  container: worldContainer,
  mode: 'dots',
  text: '出兵！',
  color: 0xffcc00,
  dots: 5,
  duration: 2000,
})

// 宣战模式（光球 + 爆炸 + 飘字）
await playArcAnimation({
  from: { x: 500, y: 300 },
  to: { x: 600, y: 400 },
  container: worldContainer,
  mode: 'orb',
  explosion: true,
  shockwaves: 3,
  text: '宣战！',
  color: 0xff4444,
  duration: 1200,
  explosionDuration: 800,
})
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| from | `{x, y}` | - | 起点屏幕坐标 |
| to | `{x, y}` | - | 终点屏幕坐标 |
| container | Container | - | PixiJS 容器 |
| mode | `'dots'\|'orb'` | `'dots'` | 动画模式 |
| explosion | boolean | `false` | 是否爆炸 |
| shockwaves | number | `3` | 冲击波层数 |
| text | string | - | 弹出文字（可选） |
| textColor | number | `0xffffff` | 弹出文字颜色 |
| highlightGfx | Graphics | - | 高亮图形对象 |
| fromFeature | Feature | - | 出发地Feature |
| toFeature | Feature | - | 目的地Feature |
| onHighlight | function | - | 高亮回调 |
| color | number | `0xffcc00` | 动画颜色 |
| dots | number | `5` | 箭头数量（dots模式） |
| spacing | number | `0.08` | 箭头间距（dots模式） |
| duration | number | `2000` | 飞行时长（毫秒） |
| explosionDuration | number | `800` | 爆炸时长（毫秒） |

### startBattleAnimation - 战斗动画（持续交火）

两城之间双向发射光点，持续循环直到手动停止。高亮在启动时绘制，战斗期间保持，停止时一并清除。

```js
import { startBattleAnimation } from '@/utils/troopAnimation'

const battle = startBattleAnimation({
  from: { x: 500, y: 300 },
  to: { x: 600, y: 400 },
  container: worldContainer,
  highlightGfx: battleHighlightGfx,
  fromFeature: featureA,
  toFeature: featureB,
  onHighlight: (feature, color) => highlightOn(battleHighlightGfx, feature, color),
  colorA: 0x3b82f6,
  colorB: 0xef4444,
  dotsA: 3,
  dotsB: 3,
  spacing: 0.15,
  speed: 0.004,
})

// 停止战斗（清除动画和高亮）
battle.stop()
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| from | `{x, y}` | - | 起点屏幕坐标 |
| to | `{x, y}` | - | 终点屏幕坐标 |
| container | Container | - | PixiJS 容器 |
| highlightGfx | Graphics | - | 高亮图形对象（可选） |
| fromFeature | Feature | - | 出发地 Feature |
| toFeature | Feature | - | 目的地 Feature |
| onHighlight | function | - | 高亮回调 `(feature, color) => void` |
| colorA | number | `0x3b82f6` | 出发方颜色（蓝色） |
| colorB | number | `0xef4444` | 目标方颜色（红色） |
| dotsA | number | `3` | A 方小球数量 |
| dotsB | number | `3` | B 方小球数量 |
| spacing | number | `0.15` | 小球间距 |
| speed | number | `0.004` | 每帧移动速度 |

**返回值：** `{ stop: () => void, graphics: Graphics }`

**注意：**
- 此函数**不返回 Promise**，不能 `await`
- 动画持续循环，需调用 `.stop()` 手动停止
- 不支持 `text` 参数，不会弹出飘字

### 弹出文字说明

`playArcAnimation` 和 `playScoutAnimation` 支持 `text` 参数，在动画结束时在目的地上方弹出飘字：
- 从目标位置上方 30px 开始
- 向上飘动 40px
- alpha 从 1 渐变到 0
- 持续约 1.2 秒后自动销毁

`startBattleAnimation` 不支持 `text` 参数。
