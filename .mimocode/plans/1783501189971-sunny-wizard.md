# Plan: 用地点 ID 替代手动画坐标调用

## 目标

三个动画函数（`playArcAnimation`、`playScoutAnimation`、`startBattleAnimation`）的 `from`/`to` 参数保持 `{x, y}` 不变，**在调用方加一个公共解析函数**，把"按 ID 找 feature → 算质心 → 转屏幕坐标"三步封装掉。

`troopAnimation.js` 零改动。

---

## 当前问题

每个动画调用都要写 4 行样板代码：

```js
const fromFeature = currentData.features.find(f => f.properties?.gb === fromGB)
const toFeature   = currentData.features.find(f => f.properties?.gb === toGB)
const from = getFeatureCentroid(fromFeature)
const to   = getFeatureCentroid(toFeature)
```

---

## 数据层的 ID 体系（已确认）

| 图层 | GeoJSON properties | ID 字段 | 示例 |
|------|-------------------|---------|------|
| 中国_市 | `name`, `gb` | `gb` | `156500000` |
| 中国_省 | `name`, `gb`, `code`, `filename` | `gb` | `156110000` |
| 世界 | `name`, `full_name`, `iso_a2`, `iso_a3`, `iso_n3` | `iso_a3` | `JPN` |

三个图层 ID 字段不同，但同一个 `currentData` 内只有一种图层，所以按顺序尝试即可。

---

## 方案：加 `resolveLocation` 辅助函数

在 `LeafletMap.vue` 中新增一个函数，靠近 `getFeatureCentroid` 旁边：

```js
/**
 * 按 ID 解析地点 → { x, y } 屏幕坐标
 * 支持 gb（中国市/省）、iso_a3（世界国家）
 * 找不到返回 null
 */
function resolveLocation(id, geoData) {
  if (!id || !geoData) return null

  // 优先在当前图层找
  let feature = geoData.features.find(f =>
    f.properties?.gb === id || f.properties?.iso_a3 === id
  )

  // 跨图层兜底：搜索所有缓存的 GeoJSON
  if (!feature) {
    for (const [, cached] of geoJsonCache) {
      feature = cached.features.find(f =>
        f.properties?.gb === id || f.properties?.iso_a3 === id
      )
      if (feature) break
    }
  }

  return feature ? getFeatureCentroid(feature) : null
}
```

> 跨图层兜底说明：当前显示中国市图层时，传世界国家 ID（如 `JPN`）也能正确解析；反过来显示世界图层时，传中国城市 ID（如 `156500000`）同样有效。`geoJsonCache` 中的数据只读，无副作用。

### 调用方变化

**派兵（`testTroopMove`）**
```js
// 改前：4 行 find + centroid
// 改后：2 行
const from = resolveLocation('156500000', currentData)
const to   = resolveLocation('156450200', currentData)
if (!from || !to) return
// fromFeature/toFeature 仍需单独获取（用于高亮），保持不变
```

**探察（`testScout`）**
```js
const from = resolveLocation('156500000', currentData)
if (!from) return
```

**宣战（`testDeclareWar`）**
```js
const from = resolveLocation('156500000', currentData)

if (!from || !to) return

```

**战斗（`testBattle`）**

```js
// 循环内：
const to = resolveLocation(target.toGB, currentData)
if (!to) continue
```

## 为什么不改 troopAnimation.js

- 动画函数的设计意图是纯坐标输入，不耦合 GeoJSON 数据结构
- `geoToScreen` 依赖 `app.screen.width/height`，这个上下文在组件层才有
- 当前 API 对调用方已经够清晰，问题只是每次调用前的"查坐标"太啰嗦
- 组件内封装一个辅助函数，改动最小、风险最低、最容易review
